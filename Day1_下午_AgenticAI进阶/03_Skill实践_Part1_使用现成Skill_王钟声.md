# 03 — Skill 实践 Part 1：使用现成 Skill（20min）— 王钟声

**形式**：讲师投屏演示 8min → 教师跟着操作 12min

---

## 环节设计

### Step 1：什么是 Skill（2min，口头讲，不另做 PPT）

> "刚才哲栋老师演示绘本生成器时，Agent 自动按'规划→文案→插图→语音→组装'的流程走完了。这个流程不是每次重新想的——它封装在 Skill 里。"
>
> **Skill = 菜谱**。你不会每次做红烧肉都重新想放什么——菜谱写好了，照着做就行。
>
> **单次 Prompt vs Skill 对比**：

| 维度 | 单次 Prompt | Skill |
|------|-----------|-------|
| 性质 | 一次性指令 | 可复用的标准流程 |
| 一致性 | 每次输出可能不同 | 每次都按同样标准 |
| 效率 | 每次重新写 | 一键触发 |
| 比喻 | 口头交代 | 书面 SOP |

> "一个 Skill 可以很简单——只有一个 SKILL.md 文件，用自然语言写。你写得越清楚，它执行越稳定。不需要写代码。"

---

### Step 2：认识绘本生成器的 2 个现成 Skill（3min，投屏展示）

王钟声打开 `.claude/skills/` 目录，展示 Skill 结构：

```
.claude/skills/
├── picturebook-planner/
│   └── SKILL.md          ← 绘本规划：输入主题→输出结构化大纲
└── page-content-writer/
    └── SKILL.md          ← 文案生成：根据大纲写每页文案+情感标签
```

> "还有两个 Skill（插图生成、网页组装）也已预装，但依赖图片 MCP。我们先跑不需要图片的部分，等下半场配好 MCP 再跑完整版。"

打开 `picturebook-planner/SKILL.md`，展示内容结构：

```markdown
---
name: 绘本规划师
description: 根据主题拆解绘本结构，生成结构化大纲
trigger: ["做绘本", "生成绘本", "picture book"]
---

# 绘本规划师

## 适用场景
需要为一个主题/课文创建绘本时使用。

## 输入要求
- 主题（必填）
- 目标年龄段（必填）
- 页数要求（可选，默认 6 页）

## 输出格式
JSON 格式：封面 + N 页正文 + 封底，每页包含标题和画面描述。

## 执行步骤
1. 根据主题确定核心信息点
2. 按年龄调整语言难度
3. 拆分为封面+正文+封底结构
4. 输出 JSON 格式大纲，等我确认
...
```

> "看到了吗？SKILL.md 就是一个 Markdown 文件。7 个部分：Frontmatter + 适用场景 + 输入要求 + 输出格式 + 执行步骤 + 示例 + 注意事项。不是代码，是自然语言说明书。"

### Step 3：讲师演示（3min）

王钟声投屏操作：

```
操作 1：检查 Skill 目录
ls .claude/skills/

操作 2：调用绘本规划 Skill
输入 prompt：
"/skill 绘本规划师
 主题：春晓（孟浩然），面向小学二年级，6页"

操作 3：观察 Agent 输出
Agent 生成 JSON 大纲：
{
  "title": "春晓",
  "cover": {"title": "春晓", "scene": "春天清晨，小鸟在枝头"},
  "pages": [
    {"id":1, "title":"春眠不觉晓", "text":"...", "scene":"..."},
    {"id":2, "title":"处处闻啼鸟", "text":"...", "scene":"..."},
    ...
  ],
  "backcover": {"title": "谢谢观看", "scene": "春天花园全景"}
}

操作 4：确认大纲后，手动调用第二个 Skill

回复"确认"，Agent 将大纲保存为 outline.json。

然后输入：
```
/skill 文案生成器
大纲文件：outline.json，目标年龄：小学二年级
```

Agent 根据大纲逐页生成文案+情感标签。

**解说**：
- "两个 Skill，两次调用。第一个出大纲，第二个写文案。"
- "Skill 的价值：SOP 写一次，每次按同样标准执行。不满意说'把第 3 页改一下'就行。"

---

### Step 4：教师任务 — 调用现成 Skill（10min）

**教师操作**：

```
1. 把 2 个 Skill 文件夹复制到自己的 .claude/skills/ 目录

2. 调用绘本规划师 Skill：
   /skill 绘本规划师
   主题：____（选自己专业的主题），面向____年级，6页

3. 观察 Agent 输出的大纲
   检查：页数对吗？每页有标题+画面描述吗？
   不满意就说"修改：____"，满意就回复"确认"

4. 调用文案生成器 Skill：
   /skill 文案生成器
   大纲文件：outline.json，目标年龄：____年级

5. 检查每页文案：字数合适吗？语言难度匹配年龄段吗？情感标签合理吗？
```

**教师各自选主题**（投影展示参考）：

| 专业 | 推荐绘本主题 |
|------|-------------|
| 汉语言 | 《静夜思》《春晓》《登鹳雀楼》、成语故事 |
| 影视 | 分镜头绘本（每页=一个镜头）、电影场景解说 |
| 音乐 | 音乐家故事、乐器介绍、歌曲意境绘本 |
| 表演 | 剧本片段绘本（角色对话+舞台指示） |
| 通用 | 课程导入绘本、知识点图解、学生行为规范 |

> 王钟声巡场，帮遇到问题的教师排错。
> 关键提示："你现在不是在一问一答——你定义目标，Agent 按 Skill 规范执行。两个 Skill，两次调用，流程就通了。"

## 魔改挑战

**初级**（做完的教师先试）：
- 把页数从 6 页改成 8 页，重新生成
- 把目标年龄从"小学二年级"改成"幼儿园大班"，观察文案变化

**进阶**：
- 在确认大纲时说"修改：把第 3 页和第 4 页交换顺序"
- 在最终预览时说"修改：背景颜色改成暖色调"
