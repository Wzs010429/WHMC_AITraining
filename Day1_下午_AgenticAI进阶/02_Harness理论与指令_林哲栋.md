## PPT 内容（5 页）

### 第 1 页：什么是 Harness

> Harness = Agent 的执行框架 / "脚手架"

**三个比喻**：

| 比喻 | 对应 |
|------|------|
| 手机操作系统（iOS/Android） | 你感受不到它，但没有它 App 跑不起来 |
| 电影导演 | 不演戏，但决定谁上场、什么时候上场、怎么配合 |
| 工厂流水线传送带 | 把半成品送到下个工位，质检员在关键工位检查 |

> Claude Code 本身就是 Harness。你们上午已经用了，现在来认识它。
>
> 刚才演示绘本生成器时，Agent 调了 30 多次工具——谁在调度这 30 多次调用？Harness。

---

### 第 2 页：Harness 的 7 层扩展体系

```
⑦ Subagents（子代理）    ← 独立上下文并行干活
⑥ MCP Servers           ← 连接外部工具与数据源（图片 API、TTS）
⑤ LSP（语言服务器）      ← IDE 级代码导航
④ Plugins               ← Skills+Hooks+MCP 打包分发
③ Skills                ← 按需加载的专业知识包（绘本 Skill）
② Hooks                 ← 会话生命周期钩子（审核点）
① CLAUDE.md             ← 项目上下文文件
────────────────────────────
   模型本身（地板）
```

> **模型能力是地板，配置质量才是天花板。**
>
> 花时间把配置做好，比追最新模型版本更有实际收益。
>
> 初学者先把前三层做扎实：**CLAUDE.md → Hooks → Skills**。

---

### 第 3 页：Harness 的 7 个核心职责

| # | 职责 | 通俗解释 | 绘本案例中的表现 |
|---|------|---------|----------------|
| 1 | 接收任务 | 读懂你输入的目标描述 | "做一个静夜思绘本" |
| 2 | 拆解步骤 | 把大目标分解成小任务 | 规划→文案→插图→语音→组装 |
| 3 | 调度工具 | 决定现在调用哪个工具 | Read Skill → Write 文件 → 调图片 API → 调 TTS |
| 4 | 传递上下文 | 把上一步结果喂给下一步 | 生成第 5 页文案时知道前 4 页内容 |
| 5 | 处理错误 | 调用失败时重试或换方案 | 图片 API 超时 → 自动重试 |
| 6 | 执行 Hook | 到审核点就停，等人操作 | 大纲生成完停一次，网页生成完停一次 |
| 7 | 记录日志 | 每一步记下来，可回溯 | 对话历史、右侧工具调用面板 |

---

### 第 4 页：Harness 怎么"思考" — LLM Loop 机制

```
用户输入任务："做个静夜思绘本"
    ↓
Harness 分析 → 需要什么工具？先去读 Skill 文件
    ↓
调用 Read 工具 → 拿到 Skill 规范
    ↓
Harness 再分析 → 现在需要生成大纲，调模型 API
    ↓
模型生成大纲 → Harness 写文件 + 触发 Hook（停！等确认）
    ↓
用户确认 → Harness 继续 → 调模型写文案 → 调图片 API → 调 TTS
    ↓
Harness 检查：6 页都完成了吗？→ 完成了，组装 HTML
    ↓
触发 Hook → 等用户预览确认 → 结束
```

> "这就是为什么 Claude Code 有时会连续调好几个工具——Harness 在做'思考→行动→观察→再思考'的循环。"

---

### 第 5 页：Claude Code 中你需要知道的指令与配置

| 指令/操作 | 作用 | 什么时候用 |
|----------|------|----------|
| 输入 prompt | 触发 Agent 开始工作 | 任何任务的开端 |
| 观察右侧工具调用日志 | 看 Harness 在调度什么工具 | 理解 Agent 在执行哪一步 |
| 回复"确认"/"继续" | 通过 Hook 审核点 | AI 停下来等你时 |
| 回复"修改：..." | 在 Hook 点修改后继续 | 对 AI 输出不满意时 |
| 把报错贴回去 | 让 Agent 自我修复 | 工具调用失败时 |
| `/skill <名称>` | 手动调用一个 Skill | 想用特定能力时 |
| `/model` | 切换模型 | 简单任务用便宜模型，复杂任务用强模型 |
| `settings.json` | 配置 Harness 行为（Hook、权限、MCP） | 定制 Agent 工作方式 |
| `CLAUDE.md` | 项目说明书，每次会话自动加载 | 让 AI 了解你的项目背景和规范 |

**三种权限模式**：

| 模式 | 行为 | 适用场景 |
|------|------|---------|
| Safe（安全） | 所有危险操作都需确认 | 新手阶段 |
| Approve（审批） | 大部分操作要确认 | 日常开发 |
| Edit（编辑） | AI 自主操作 | 完全信任 AI |

## 现场演示脚本（8min）：绘本生成器的工具调用日志

> 林哲栋重新跑一遍绘本生成器，这次不是看成品，而是逐步骤**盯着右侧工具调用日志**讲解。

### Step 1（1min）：输入任务 + 观察初始调度

```
输入："帮我规划一个绘本：主题'春晓'，面向小学二年级，6页，先生成大纲等我确认"
```

**解说**：
- "看右侧面板——Harness 第一个动作是 `Read`。读什么？读 Skill 文件。"
- "Harness 在'学习'——它要知道绘本规划的规范是什么。这是**职责 3：调度工具**。"

### Step 2（2min）：观察 Skill 加载 + 模型调用

```
工具调用日志：
[Read]  .claude/skills/picturebook-planner/SKILL.md
[API]   DeepSeek V4 → 生成大纲
[Write] outline.json
```

**解说**：
- "Read 之后是 API 调用——Harness 判断'规范有了，该让模型干活了'。"
- "模型输出后，Harness 自动 Write 文件——不是你在写，是 Harness 在写。**职责 1+2+3** 一气呵成。"
- "然后停了。为什么？你说'等我确认'——**职责 6：执行 Hook**。"

### Step 3（1min）：确认后继续 — LLM Loop 演示

```
操作：回复"确认"
```

```
工具调用日志：
[API]   DeepSeek V4 → 第1页文案
[API]   图片 MCP → 生成第1页插图
[Write] page1.png
[API]   DeepSeek V4 → 第2页文案
[API]   图片 MCP → 生成第2页插图
[Write] page2.png
...（循环 6 次）
```

**解说**：
- "看到循环了吗？**职责 4：传递上下文**——生成第 3 页时，前 2 页的内容自动带进 prompt。"
- "你不需要每次说'记得前面写了什么'——Harness 帮你记住了。"

### Step 4（2min）：观察 TTS 调用 + 组装

```
工具调用日志：
[API]   TTS MCP → 第1页旁白 <|emotion:calm|>
[Write] page1.mp3
[API]   TTS MCP → 第4页旁白 <|emotion:longing|>
[Write] page4.mp3
...
[Write] picturebook.html
```

**解说**：
- "TTS 调用是通过 MCP——**职责 3：调度工具**，Harness 不关心工具是哪来的，有 MCP 协议就能调。"
- "最后 Write HTML——全部完成后触发 Hook，停，等人确认。"
- "整个流程 Harness 调了约 30-40 次工具。你只操作了 2 次——确认大纲、确认预览。"

### Step 5（2min）：指令速查时间

**解说**（回到 PPT 第 5 页）：

> "最后 2 分钟，记住这几个最有用的操作："
> 1. "**看右侧日志**——理解 Agent 在做什么。不要只看聊天框。"
> 2. "**确认/继续/修改**——Hook 停住时这三个词最有力量。'修改：把第 3 页改暗一点'。"
> 3. "**把报错贴回去**——工具调用失败不要慌，贴给 Agent 就行。"
> 4. "**CLAUDE.md**——花 5 分钟写清楚你的项目，比追最新模型有用。（模型是地板，配置是天花板）"

## 关键教学提示

1. **演示时右侧面板要始终可见**。教师们第一次意识到"原来 AI 不是在魔法，是在一步步调工具"。
2. **Step 3 的循环是核心**。教师要看着 Agent 自动跑 6 轮（文案→插图→保存），理解 LLM Loop。
3. **不要同时讲 MCP 概念**。TTS 和图片 API 调用时说"这是通过 MCP 调的，后面会讲"，不展开。
4. **指令速查表可以打印出来**，发给教师贴在桌上。下午全程会反复用到。
