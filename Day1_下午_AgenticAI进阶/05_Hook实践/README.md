# 05 — Hook 纯实践（27min）— 林哲栋

**形式**：不讲 PPT，直接动手配、动手跑

---

## 环节设计

### Step 1：交代背景（1min）

> "刚才王老师带你们跑了绘本生成器、还创建了自己的 Skill。Agent 在大纲生成后自动停了——那就是一个 Hook。现在我们要在自己创建的 Skill 流程里面，也加上这种'自动暂停等确认'的机制。"

---

### Step 2：理解 Hook — 一张简图（3min）

林哲栋画一张简图：

```
Agent 绘本工作流：
[规划大纲] → [写文案] → [Hook: 停！大纲对不对？] → [生成插图] → [TTS配音] → [组装网页] → [Hook: 停！预览满不满意？] → [输出]
```

> "Hook 就是 AI 工作流中的'安检口'——到了关键节点，必须等人确认才能继续。"

**三种 Hook 类型**：

| 类型 | 触发时机 | 绘本案例 |
|------|---------|----------|
| **PreToolUse** | 工具调用**之前** | AI 要调图片 API 了，先问你"可以调吗？" |
| **PostToolUse** | 工具调用**之后** | AI 写完文件了，自动弹提醒"插图已生成，建议检查" |
| **Human-Review** | 关键决策点 | 大纲生成后、网页生成后，必须等你确认 |

> "Hook 放在哪里，取决于'哪一步做错了后果最严重'。后果越严重，越要靠前放 Hook。"

---

### Step 3：配置第一个 Hook — PostToolUse 提醒（8min）

林哲栋投屏演示，教师跟着操作：

**操作 1**：打开 `.claude/settings.json`

**操作 2**：加入 Hook 配置：

```json
{
  "permissions": {
    "allow": [
      "Bash(npm *)",
      "Bash(git *)",
      "Bash(python *)",
      "Bash(llama-*)"
    ]
  },
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "echo \"[Hook提醒] AI 刚修改了文件，建议检查一下改了什么\""
          }
        ]
      }
    ]
  }
}
```

**解说**：
- "`matcher: Write|Edit` 表示：每次 AI 写文件或编辑文件之后触发。"
- "`command` 是在终端执行一条命令——这里是打印一条提醒。"
- "你也可以改成：自动备份、自动格式化、自动检查语法。"

**操作 3**：教师在自己的 settings.json 里加上这段配置。

**操作 4**：重新跑绘本生成器（或刚才写的 Skill），观察效果：

> 每次 Agent 写文件后，终端弹出 `[Hook提醒] AI 刚修改了文件，建议检查一下改了什么`

---

### Step 4：配置强 Hook — 人工审核拦截（7min）

> "刚才的 Hook 只是提醒。现在做真正的拦截——大纲生成后、网页生成前，必须等你点确认。"

**操作**：不用改 JSON，直接在 prompt 中指定审核点：

```
"帮我做绘本：主题'登鹳雀楼'，面向小学四年级，6页。

审核点 1：大纲生成后必须停下等我确认，我确认后才能继续写文案。
审核点 2：所有内容写完后、最终网页生成前，必须停下等我确认。

每页都要配插图+语音旁白。"
```

教师跑这个流程，体验 Agent **两次停下来等确认**：

```
Agent: 大纲已生成，请确认：
{
  "title": "登鹳雀楼",
  "pages": [...]
}
→ 教师：确认，继续

（Agent 自动跑完文案→插图→TTS）

Agent: 内容全部生成完毕，准备组装网页。请确认是否满意？
→ 教师：第 3 页的背景太暗了，改成暖色调。确认后继续。
```

**解说**：
- "在 prompt 里直接写'审核点：...停下等我确认'——Agent 理解这句话后，会在流程中主动暂停。"
- "JSON 配置的 Hook 是系统级拦截（每次特定工具调用后自动触发），Prompt 里的审核点是 Agent 行为级暂停（Agent 理解你的意图后主动停）。"
- "两种方式互补：全局规则用 JSON，任务级审核点写 prompt。"

---

### Step 5：在自己创建的 Skill 里加 Hook（8min）

**任务**：在刚才自己创建的 Skill 中，找合适的节点加上审核 Hook。

**引导思考**（林哲栋巡场时提示）：

> "你的论文批改 Skill——在'给出评分'之前，要不要加 Hook？"
>
> "你的讨论题生成 Skill——在'发布给学生'之前，要不要加 Hook？"

**操作**：在 Skill 的"执行步骤"中，手动加入审核点：

```markdown
## 执行步骤
1. 读取学生论文
2. 按评分标准逐项评分
3. **Hook 审核点：评分结果必须等教师确认后才能继续**
4. 给出修改建议
5. 生成评语
6. **Hook 审核点：评语必须等教师确认后才能输出**
```

或者直接在调用 Skill 时加 Hook：

```
"/skill 论文批改助手
 论文：[上传文件]
 **评分结果生成后必须停下等我确认**
 **评语输出前必须停下等我确认**"
```

## 魔改挑战

**初级**：
- 把 Hook 提醒文字改成个性化的（如 "[王老师提醒] 请检查"）
- 多加一个 Hook 审核点（如"插图生成后确认"）

**进阶**：
- 尝试 PreToolUse Hook：在 AI 调用图片 API 之前拦截，问"是否继续？"
- 修改命令：让 Hook 不只是提醒，而是自动备份修改过的文件（`cp $FILE $FILE.bak`）
