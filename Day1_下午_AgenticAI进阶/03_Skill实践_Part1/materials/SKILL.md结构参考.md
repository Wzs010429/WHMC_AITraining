# SKILL.md 结构参考（7 个部分）

```markdown
---
name: 你的Skill名称
description: 一句话描述这个Skill做什么
trigger: ["触发关键词1", "触发关键词2"]
---

# Skill名称

## 适用场景
什么时候应该用这个Skill？

## 输入要求
需要什么信息？

## 输出格式
输出应该长什么样？

## 执行步骤
1. 第一步
2. 第二步
3. ...

## 示例
一个具体的输入→输出示例

## 注意事项
有哪些坑要避免？
```

## 各部分说明

| 部分 | 必填 | 说明 |
|------|:--:|------|
| Frontmatter（name, description, trigger） | ✅ | Claude 用来判断何时触发此 Skill |
| 适用场景 | ✅ | 描述"什么时候该用" |
| 输入要求 | ✅ | 需要什么信息才能运行 |
| 输出格式 | ✅ | 输出应该长什么样（JSON/Markdown/文本） |
| 执行步骤 | ✅ | **Skill 的灵魂**——想清楚分几步，每步做什么 |
| 示例 | 推荐 | 一个完整的输入→输出例子 |
| 注意事项 | 推荐 | 常见坑、要注意的地方 |

## 关键原则

- 用自然语言写，不需要写代码
- 写得越清楚，执行越稳定
- 执行步骤是灵魂——想清楚你的任务分几步，每一步做什么
- Description 写给模型看，不要写营销文案
