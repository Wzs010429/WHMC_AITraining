# 07 — 总结（10min）— 王钟声

**形式**：成果展示 3min + 互动回顾 5min + 衔接第二天 2min

---

## 环节设计

### 快速展示（3min，严格限时）

请 1-2 位教师投屏展示（每人不超过 1.5min，讲师用计时器）：

1. **自己创建的 Skill**（SKILL.md 内容，10 秒扫一眼）
2. **加了 Hook 的流程**（在哪个节点停的？20 秒）
3. **带语音+插图的最终绘本网页**（浏览器打开，翻页，播放语音，1min）

> "用一句话说出：你今天下午做了什么？"

---

### 一张图回顾全天（5min，互动问答）

> **不要独白。指着图问，让教师集体回答。每层停顿 5 秒等回答。**

```
                    ┌────────────────────────────────┐
                    │         Agent（智能体）          │
                    │   模型 + 工具 + 流程 + 记忆 + 审核  │
                    │                                │
                    │  ┌──────────────────────────┐  │
                    │  │   Harness（Claude Code）   │  │
                    │  │   调度 | 指令 | 5层扩展     │  │
                    │  │   模型是地板，配置是天花板    │  │
                    │  └──────────┬───────────────┘  │
                    │             │                  │
                    │  ┌──────────▼───────────────┐  │
                    │  │ Skill 层                   │  │
                    │  │ superpowers | frontend-design  │  │
                    │  │ + 你自己创建的 Skill        │  │
                    │  └──────────┬───────────────┘  │
                    │             │                  │
                    │  ┌──────────▼───────────────┐  │
                    │  │ Hook 层（人工审核点）        │  │
                    │  │ PreToolUse | PostToolUse  │  │
                    │  │ 大纲确认 | 最终预览         │  │
                    │  └──────────┬───────────────┘  │
                    │             │                  │
                    │  ┌──────────▼───────────────┐  │
                    │  │ MCP 层（连接外部世界）       │  │
                    │  │ 图片API(FLUX) | TTS(Higgs) │  │
                    │  │ AI 世界的 USB Type-C       │  │
                    │  └──────────────────────────┘  │
                    └────────────────────────────────┘
```

**互动问答顺序**（指着每一层问，不等举手，集体喊出答案）：

> 指 Agent 层 → "Agent 有哪五个要素？"（模型、工具、流程、记忆、审核）
> 指 Harness 层 → "谁在背后调度这 30 多次工具调用？"（Harness / Claude Code）
> 指 Skill 层 → "Skill 的比喻是什么？"（菜谱）
> 指 Hook 层 → "Hook 的比喻是什么？为什么需要它？"（安检口 / AI 干活人把关）
> 指 MCP 层 → "MCP 的比喻是什么？今天连了哪两个？"（USB Type-C / 图片 + TTS）

> "这 5 层，你今天全部动手操作过。从一句 prompt 到一个有声绘本——每一层你都亲手碰过。"
>
> 将三原则融入回顾中，不再单独讲：
> -（指 Hook 层）"所以 Agent 不是全自动的——Hook 让'人说了算'"
> -（指 Skill 层）"你的教学经验封装成 Skill，就是你的 AI 助手"
> -（指 MCP 层）"MCP 即插即用——今天接图片和语音，明天可以接翻译、搜索"

---

### 衔接第二天

> "今天你们用 Agent 做了一个有声绘本。明天学什么？"
>
> - **明天上午**（Benchmark / 数据）— 怎么系统性评测绘本质量？一百个学生用了你的绘本，怎么知道好不好？
> - **明天下午**（多智能体 / RAG）— 多个 Agent 协作：一个 Agent 写文案、一个 Agent 画插图、一个 Agent 审核。加上知识库检索，就是一个完整的**智能课程系统**。
>
> "今天教会 AI 怎么做事。明天教会 AI 怎么把事做得更好。"

---

## 教师自检清单（课后发给教师）

- [ ] 我能用自己的话说出 Agent 的五个要素
- [ ] 我能解释 LLM Loop（感知→推理→行动→反馈）
- [ ] 我会看 Claude Code 右侧的工具调用日志
- [ ] 我调用过现成 Skill，跑通了完整流程
- [ ] 我创建了 1 个自定义 Skill
- [ ] 我在流程中配置了至少 1 个 Hook
- [ ] 我配置了图片 MCP + TTS MCP
- [ ] 我产出了 1 个带语音+插图的绘本网页
