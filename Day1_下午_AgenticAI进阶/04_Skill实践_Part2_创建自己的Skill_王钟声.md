## 环节设计

### Step 1：Skill 写法 — 模板讲解（3min）

王钟声展示空白 Skill 模板：

```markdown
---
name: 你的Skill名称
description: 一句话描述这个Skill做什么
trigger: ["触发关键词1", "触发关键词2"]
---

# Skill名称

## 适用场景
什么时候应该用这个Skill？

## 输入要求
需要什么信息？

## 输出格式
输出应该长什么样？

## 执行步骤
1. 第一步
2. 第二步
3. ...

## 示例
一个具体的输入→输出示例

## 注意事项
有哪些坑要避免？
```

> "7 个部分，填完就是一个完整的 Skill。关键是 **执行步骤** 和 **输出格式** 要写清楚——这两部分决定了 Agent 执行的质量。"
>
> "不需要写代码——用自然语言写就行。你写得越清楚，它执行越稳定。"

### Step 2：讲师现场示范 — 快速创建一个 Skill（2min）

王钟声对 Claude Code 说：

```
"帮我创建一个 Skill，叫'唐诗绘本生成器'。
功能：输入一首唐诗，自动生成 6 页儿童绘本。
执行步骤：
1. 先解析诗歌含义（每句对应什么画面）
2. 确定目标年龄段的语言难度
3. 生成 6 页文案，每页 2-3 句，语言符合年龄
4. 为每页选情感标签（Higgs TTS 21种情感）
5. 为每页写画面描述（供图片 API 使用）
6. 输出完整 JSON

输出格式：
{
  poem: '原诗',
  age: '年龄段',
  cover: {title, scene},
  pages: [{id, title, text, emotion, scene_desc}],
  backcover: {title, scene}
}

注意事项：
- 文案不要直接抄原诗，要用孩子能懂的话改编
- 情感标签用英文（calm, awe, longing 等）
- 画面描述要具体（物件、颜色、构图），不能抽象"
```

Claude Code 自动生成 SKILL.md 文件。王钟声展示：
- 文件位置：`.claude/skills/tang-poem-picturebook/SKILL.md`
- Frontmatter 已填充
- 7 个部分都已完成
- 如果某部分不满意："帮我把执行步骤第 3 步改一下"

> "2 分钟，一个有 6 步执行流程的 Skill 就有了。这就是 Vibe Coding + Skill——你描述需求，AI 帮你封装。不满意继续改。"

### Step 3：教师动手 — 每人创建一个 Skill（8min）

**任务**：为你的教学场景创建一个 Skill。

**操作方式**（精简版，直接 copy 模板填空）：

```
"帮我创建一个 Skill，叫____。
功能：____。
输入：____。
输出：____。
执行步骤：
1. ____
2. ____
3. ____"
```

**参考方向**（投影展示，教师直接选一个）：

| 场景 | Skill 名称 | 功能描述 |
|------|-----------|----------|
| 论文批改 | 论文批改助手 | 输入学生论文 → 按评分标准打分 → 标出问题段落 → 给出修改建议 |
| 讨论题生成 | 讨论题设计师 | 输入课程内容 → 生成 3 道开放式讨论题 → 每题附参考答案要点 |
| 错题分析 | 错题分析师 | 输入学生错题 → 分类错误类型 → 生成针对性练习 |
| 课程大纲 | 课程大纲规划师 | 输入课程名+课时 → 拆解章节 → 分配学时 → 标注重难点 |
| 行业简报 | 行业动态简报 | 输入行业关键词 → 搜索最新新闻 → 汇总简报 |
| 教学案例 | 教学案例生成器 | 输入知识点 → 生成 3 个生活化案例 → 附讨论问题 |

> 王钟声巡场，帮教师梳理思路。
> 关键提示："你不需要写出完整的 Skill 文件——把需求说清楚，AI 帮你写。写完看看，不满意继续让它改。"
> "**执行步骤**是 Skill 的灵魂——想清楚你的任务分几步，每一步做什么。"

## 魔改挑战

**初级**：
- 在 Skill 的"注意事项"里加一条你踩过的坑
- 改一下 trigger 关键词，让它更容易被触发

**进阶**：
- 给 Skill 加一个"示例"部分——写一个完整的输入→输出例子
- 让 Agent 帮你把 Skill 的"执行步骤"从 3 步扩展到 5 步，更细致
