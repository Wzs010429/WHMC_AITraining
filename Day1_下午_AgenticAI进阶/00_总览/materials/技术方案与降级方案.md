# Day1 下午 — 技术方案与降级方案

## 技术栈

| 层 | 选型 | 部署位置 |
|----|------|----------|
| LLM | DeepSeek V4 Pro | API 调用 |
| 文生图 | FLUX.2-klein-9B 蒸馏 BF16 | L20 服务器 × 2，Aquiles-Image FastAPI |
| TTS | Higgs TTS v3 BF16 | 每台教师电脑本地 llama.cpp（~10GB 显存） |
| 调度框架 | Claude Code | 每台教师电脑 |

## 降级方案

| 组件 | 主方案 | 降级方案 |
|------|--------|----------|
| 图片生成 | L20 服务器 FLUX.2-klein-9B | 硅基流动 API（FLUX.1-schnell） |
| TTS | 本地 Higgs TTS v3 BF16 | Edge TTS MCP |
| 图片 MCP | 自写转发 L20 服务器 | aleph-siliconflow-mcp |
| TTS MCP | 自写转发本地 llama.cpp | edge-tts-mcp |

## 风险预案

| 风险 | 应对 |
|------|------|
| Agent 理论 PPT 讲太久 | "多智能体协作"页可压缩或跳过；演示已前置，PPT 压力减小 |
| Harness 7 层讲太久 | 已精简为 5 层，Plugins/LSP/Subagents 折叠为"进阶" |
| L20 图片服务大面积故障 | 降级为硅基流动 API（提前注册 Key） |
| 本地 TTS 模型没装好 | 降级为 Edge TTS MCP |
| 教师想不出 Skill 做什么 | 提供 6 个模板方向 |
| Hook JSON 格式写错 | 提供正确模板，复制粘贴即可 |
| 整体超时 | 优先确保 Agent 演示+理论 + Skill Part1 + MCP 跑通 |
| 教师进度不同步 | 关键节点有 Pulse Check（举手确认），巡场讲师针对性辅导 |
