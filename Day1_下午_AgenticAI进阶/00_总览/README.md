## 四、主 Demo：AI 绘本生成器

教师输入一个主题 → Agent 自动完成全流程：

```
输入："给三年级讲《静夜思》"
        ↓
  ① 拆解绘本结构（封面 + 4-6 页 + 封底）
        ↓
  ② 生成每页文案（适合目标年龄段）
        ↓
  ③ 调用 FLUX.2-klein-9B → 每页插图
        ↓
  ④ 调用 Higgs TTS v3 → 每页旁白语音（带情感标签）
        ↓
  ⑤ 组装成交互式翻页网页（图文 + 播放按钮）
        ↓
  ⑥ 浏览器打开 → 完整有声绘本
```

教师在 ①（大纲确认）和 ⑥（最终预览）各确认一次，中间全自动执行。

> **重要**：图片 MCP 和 TTS MCP 在课前已预配到每台教师机。教师在 Skill Part1 阶段先跑规划+文案（不依赖 MCP），到 MCP 环节（下半场）才打开完整流水线，跑通图片+语音。林哲栋的演示机预配了全部环境。

## 五、演示 vs 实践对照

| 环节 | 林哲栋演示（投屏） | 教师动手 |
|------|-------------------|----------|
| 🔥 开场演示 | 跑完整绘本流程（预配环境），先惊艳 | 观察，脑中建立画面 |
| Agent 理论 PPT | 回到 PPT 解释"刚才发生了什么" | 带着 Demo 画面理解概念 |
| Harness | 演示工具调用日志 | 观察右侧面板，理解 Harness 调度 |
| Skill Part1 | 展示 2 个真实社区 Skill（superpowers + frontend-design） | 安装并使用 superpowers 规划教学项目，体验 frontend-design |
| Skill Part2 | 1.5 分钟现场创建一个 Skill | 每人创建 1 个自定义 Skill（10.5min） |
| Hook | 展示审核点拦截效果 | 在自己流程中加 Hook |
| MCP | 一次性展示两个 MCP Server 配置 | 理解配置，跑通完整流程（含插图+语音） |

## 八、风险预案

| 风险 | 应对 |
|------|------|
| Agent 理论 PPT 讲太久 | "多智能体协作"页可压缩或跳过；演示已前置，PPT 压力减小 |
| Harness 7 层讲太久 | 已精简为 5 层，Plugins/LSP/Subagents 折叠为"进阶" |
| L20 图片服务大面积故障 | 降级为硅基流动 API（提前注册 Key） |
| 本地 TTS 模型没装好 | 降级为 Edge TTS MCP |
| 教师想不出 Skill 做什么 | 提供 6 个模板方向（论文批改/讨论题生成/错题分析/课程大纲/行业简报/教学案例） |
| Hook JSON 格式写错 | 提供正确模板，复制粘贴即可 |
| 整体超时 | 优先确保 Agent 演示+理论 + Skill Part1 + MCP 跑通；Hook 的 Step 3（复制粘贴）可加速 |
| 教师进度不同步 | 关键节点有 Pulse Check（举手确认），巡场讲师针对性辅导掉队教师 |
