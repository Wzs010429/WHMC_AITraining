# 04 — Skill 实践 Part 2：创建自己的 Skill（18min）— 王钟声

**形式**：模板讲解 2min → 讲师演示 1.5min → 教师创建 10.5min → 分享 2min + 弹性缓冲 2min

## 环节设计

### Step 1：Skill 写法 — 快速回顾模板（2min）

> "刚才你们用了 2 个 GitHub 上最火的真实 Skill——superpowers（27K ⭐，自动头脑风暴+任务规划）和 frontend-design（Anthropic 官方，让网页不再千篇一律）。你们已经见过 SKILL.md 长什么样了。7 个部分，用自然语言写，不需要写代码。"

王钟声快速展示空白 Skill 模板（投屏，30 秒扫一遍）：

```markdown
---
name: 你的Skill名称
description: 一句话描述这个Skill做什么
trigger: ["触发关键词1", "触发关键词2"]
---

# Skill名称

## 适用场景
什么时候应该用这个Skill？

## 输入要求
需要什么信息？

## 输出格式
输出应该长什么样？

## 执行步骤
1. 第一步
2. 第二步
3. ...

## 示例
一个具体的输入→输出示例

## 注意事项
有哪些坑要避免？
```

> "7 个部分，填完就是一个完整的 Skill。关键是 **执行步骤** 和 **输出格式** 要写清楚——这两部分决定了 Agent 执行的质量。不需要写代码——用自然语言写就行。"

### Step 2：讲师现场示范 — 边讲边做（1.5min）

王钟声对 Claude Code 说（精简版，只演示核心 6 个字段）：

```
"帮我创建一个 Skill，叫'唐诗绘本生成器'。
功能：输入一首唐诗，自动生成 6 页儿童绘本。
执行步骤（6步）：解析诗歌→确定语言难度→生成文案→选情感标签→写画面描述→输出JSON。
输出格式：{poem, age, cover, pages: [{id, title, text, emotion, scene_desc}], backcover}
注意事项：文案用孩子能懂的话改编；情感标签用英文（calm, awe, longing等）；画面描述要具体（物件、颜色、构图）。"
```

Claude Code 自动生成 SKILL.md。王钟声快速展示生成结果（30 秒）并强调：
- 文件位置：`.claude/skills/tang-poem-picturebook/SKILL.md`
- 如果不满意某部分："帮我把执行步骤第 3 步改一下"
- "1 分半钟，一个有 6 步执行流程的 Skill 就创建好了。接下来你们来。"

---

### Step 3：教师动手 — 每人创建一个 Skill（10.5min）

**任务**：为你的教学场景创建一个 Skill。

**操作方式**（精简版，直接 copy 模板填空）：

```
"帮我创建一个 Skill，叫____。
功能：____。
输入：____。
输出：____。
执行步骤：
1. ____
2. ____
3. ____"
```

**参考方向**（投影展示，教师直接选一个）：

| 场景 | Skill 名称 | 功能描述 |
|------|-----------|----------|
| 论文批改 | 论文批改助手 | 输入学生论文 → 按评分标准打分 → 标出问题段落 → 给出修改建议 |
| 讨论题生成 | 讨论题设计师 | 输入课程内容 → 生成 3 道开放式讨论题 → 每题附参考答案要点 |
| 错题分析 | 错题分析师 | 输入学生错题 → 分类错误类型 → 生成针对性练习 |
| 课程大纲 | 课程大纲规划师 | 输入课程名+课时 → 拆解章节 → 分配学时 → 标注重难点 |
| 行业简报 | 行业动态简报 | 输入行业关键词 → 搜索最新新闻 → 汇总简报 |
| 教学案例 | 教学案例生成器 | 输入知识点 → 生成 3 个生活化案例 → 附讨论问题 |

> 王钟声巡场，帮教师梳理思路。
> 关键提示："你不需要写出完整的 Skill 文件——把需求说清楚，AI 帮你写。写完看看，不满意继续让它改。"
> "**执行步骤**是 Skill 的灵魂——想清楚你的任务分几步，每一步做什么。"

### Step 4：快速分享（2min）

请 1-2 位教师口头分享：
1. 创建了什么 Skill？
2. 执行步骤分了几步？
3. 有什么感受？

> "你们刚才创建的 Skill，下午 Hook 环节会用到——我们在 Skill 流程里面加审核点。"

- [ ] 执行步骤至少有 3 步

---

## 魔改挑战

**初级**：
- 在 Skill 的"注意事项"里加一条你踩过的坑
- 改一下 trigger 关键词，让它更容易被触发

**进阶**：
- 给 Skill 加一个"示例"部分——写一个完整的输入→输出例子
- 让 Agent 帮你把 Skill 的"执行步骤"从 3 步扩展到 5 步，更细致

---

## 阶段小结 + 衔接休息

> 💡 **Pulse Check**："SKILL.md 文件生成成功的举下手？"（5 秒快速确认）

> "你们刚才创建的 Skill，休息后的 Hook 环节会用到——我们在 Skill 流程里面加审核点。"
>
> ⏰ **休息弹性处理**：
> - 如果教师正投入创建 Skill（15:30 还没完成），可以说"大家继续做，需要休息的自行休息，15:50 准时回来。Hook 环节会从你们刚才创建的 Skill 开始。"
> - 如果提前完成（15:25 左右），引导教师做魔改挑战，或互相看对方的 Skill
> - 提前完成的教师可以趁休息时间继续完善自己的 Skill

> "休息 20 分钟。回来后林哲栋老师带你们在刚才创建的 Skill 里加 Hook——就是那种'自动停下来等你确认'的审核点。"
