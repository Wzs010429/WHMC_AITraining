# 01 — Agent 理论（30min）— 林哲栋

**形式**：🔥 演示先行 5min → PPT 讲授 20min → 逐点对应 5min

> **设计思路**：午饭后教师容易犯困。先展示成品制造"哇"的时刻抓住注意力，再回到 PPT 解释"刚才发生了什么"。每个概念教师脑中已有具体画面，对号入座即可。

## 🔥 开场演示脚本（5min，精确到秒）— 先惊艳，再解释

> 林哲栋投屏，用绘本生成器跑完整流程。**不讲概念，只展示效果。**
> **演示前说明（15秒）**："大家先看一个东西。我这台机器已经配好了，你们的环境也已预装，下午后半段我们都会打开看。先看效果——"

### 0:00–0:30 | 输入任务

```
操作：在 Claude Code 中输入 prompt
内容："帮我做一个绘本，主题'静夜思'，面向小学三年级。
      6页 + 封面封底。每页配插图+语音旁白。
      先生成大纲等我确认。"
```

**解说**："看，我只说了一句话——没说每一步怎么做。"

### 0:30–1:15 | Agent 拆解结构

Agent 自动调用 Skill，生成绘本大纲（JSON 格式）：

```
封面：月光下的诗人
第1页：床前明月光 — 诗人看见窗外的月光
第2页：疑是地上霜 — 月光像白霜铺满地
第3页：举头望明月 — 诗人抬头看月亮
第4页：低头思故乡 — 诗人想家了
第5页：远方的家乡 — 家乡的风景
第6页：诗意总结 — 月光连接着家和远方
封底：谢谢观看
```

**解说**："注意——Agent 停了。大纲生成后它在等你确认。我不点确认它不会继续。"

### 1:15–1:45 | 教师确认，Agent 继续

```
操作：回复"确认"
```

Agent 开始逐页生成：写文案 → 调图片 API → 保存插图 → 写文案 → 调图片 API → ...依次到第 6 页。

**解说**："看，Agent 在自己循环——写文案→画图→保存→下一页。不是我操作的，是它自己。"

### 1:45–2:15 | TTS 语音合成

Agent 调用 TTS，为每页生成语音旁白：

```
第1页旁白："床前明月光，诗人看见窗外的月光洒落" [情感: 平静]
第4页旁白："诗人低下头，想起了远方的家乡"    [情感: 思念]
```

**解说**："它自己为每页选了情感标签——第1页是'平静'，第4页是'思念'。"

### 2:15–3:00 | 组装网页

Agent 自动生成 HTML 文件——翻页效果 + 每页图文 + 播放按钮。浏览器打开预览。

**解说**："一个完整的有声绘本。从输入一句话到看到成品，3 分钟。我只操作了 2 次——确认大纲、确认预览。其他全是 Agent 自己做的。"

### 3:00–5:00 | 留悬念，不解释

**解说**（关键！不要在这里逐点对应，留悬念给 PPT）：

> "刚才这 3 分钟发生了什么？为什么 Agent 能自己规划、自己写文案、自己画图、自己配音、自己组装网页？它是怎么'思考'的？接下来 20 分钟，我们把刚才看到的一层一层拆开。"

**直接切到 PPT 第 1 页。**

### 第 4 页：对比案例 — 同一任务，两种模式

对比案例贯穿全程：**"做一个《静夜思》绘本，含 6 页插图 + 语音旁白 + 翻页网页"**

| 步骤 | 无 Agent（上午模式） | 有 Agent（Agentic AI） |
|------|---------------------|----------------------|
| 规划结构 | 自己写 prompt → 等输出 → 检查 → 不满意再改 | Agent 自主生成，按 Skill 标准格式输出 |
| 生成文案 | 一页一页贴 prompt → 复制 → 组装 | Agent 自动逐页生成，记住前后文 |
| 生成插图 | 每页单独描述画面 → 贴到图片工具 → 下载 | Agent 自动调用图片 API，每页配图 |
| 语音合成 | 每页文案贴到 TTS → 下载 mp3 → 改名 | Agent 自动调 TTS，带情感标签 |
| 组装网页 | 自己写/贴 HTML → 手动引入 CSS/JS | Agent 自动组装翻页网页 |
| 修改 | 找到对应位置 → 重新 prompt | "把第 3 页改暗一点"，Agent 定位修改 |
| **总操作次数** | **~40-60 次** | **2-3 次（只在审核点确认）** |
| **总耗时** | **2-3 小时** | **10-15 分钟** |

> "同样做一个绘本，上午模式要来回操作 50 次，Agent 模式你只需要确认两次。这就是今天要学的。"

---

### 第 5 页：什么是 Agent（智能体）

> Agent（智能体）= 能自主规划、执行多步骤任务、调用工具、在关键节点等待人工确认的 AI 系统。

**核心比喻**：

| 模型调用（上午） | Agent（下午） |
|---------|-------|
| 打电话问路："师傅，去故宫怎么走？" → 师傅告诉你 → 挂了 | 请了代驾："送我去故宫" → 司机自己导航、看路况、选路线、到地方停车 |
| 你问一句，它答一句 | 你定目标，它自己走，关键节点问你 |

> 回到绘本案例：上午模式是"你问师傅每步怎么走"；Agent 模式是"你上车说目的地，他开到终点"。

---

### 第 6 页：Agent vs 普通 AI 聊天的区别

| 维度 | 普通 AI 聊天 | Agent（智能体） |
|------|-----------|----------------|
| 行为 | 你问一句，它答一句 | 你说一个目标，它自己规划并执行多个步骤 |
| 能力 | 只能生成文字 | 能读写文件、运行命令、搜索、调用外部 API |
| 主动性 | 被动回答 | 主动规划、主动发现问题 |
| 记忆 | 仅限当前对话 | 可以有长期记忆 |
| 比喻 | 百科全书 | 实习助理 |

> 回到绘本案例：普通 AI 聊天只能告诉你"怎么做绘本"；Agent 直接帮你"做出绘本"。

---

### 第 7 页：Agent 的工作循环 — LLM Loop

```
       ┌─────────────────────────────────┐
       │                         │
       ▼                         │
   感知（读取文件、理解需求）         │
       │                         │
       ▼                         │
   推理（分析问题、制定计划）         │
       │                         │
       ▼                         │
   行动（修改文件、调用工具、执行命令）  │
       │                         │
       ▼                         │
   反馈（检查结果、发现新问题）──────────┘

   这个循环一直跑到任务完成。
```

> "这就是 Agent 和聊天机器人的本质区别——Agent 不是'答完就结束'，是'做完才结束'。"

> 回到绘本案例：Agent 规划完大纲（感知）→ 分析需要几页（推理）→ 调 API 生成（行动）→ 检查图片是否合理（反馈）→ 发现第 4 页画面不对，重新生成（回到感知）……一直循环到全部完成。

### 第 8 页：Agent 的五个核心要素

```
        ┌──────────────┐
        │    模型       │  ← 大脑：理解意图、做判断、生成内容
        │ (DeepSeek V4) │
        └──────┬───────┘
               │
   ┌───────────┼───────────┐
   │           │           │
   ▼           ▼           ▼
┌──────┐  ┌──────┐  ┌──────────┐
│ 工具  │  │ 流程  │  │   记忆    │
│ (手)  │  │(步骤) │  │ (笔记本)  │
└──────┘  └──────┘  └──────────┘
               │
               ▼
        ┌──────────────┐
        │   人工审核     │  ← 把关：关键决策必须人确认
        └──────────────┘
```

> 回到绘本案例：
> - **模型**= DeepSeek V4 负责写文案、选情感标签
> - **工具**= 调图片 API（FLUX.2-klein-9B）、调 TTS（Higgs TTS v3）、写文件
> - **流程**= 先规划→再写文案→再画图→再配音→最后组装
> - **记忆**= 出第 5 页时记得第 1 页写了什么
> - **审核**= 大纲确认、最终预览，两个关键节点等你拍板

---

### 第 9 页：要素 ① 模型（大脑）+ ② 工具（手）

- **模型**：负责"想"——DeepSeek V4 规划绘本结构、写文案、为每页选情感标签
- **工具**：负责"做"——调图片 API（FLUX）、调 TTS API（Higgs）、写 HTML 文件
- 关键认知：**模型 ≠ Agent**。模型只决定"智力上限"，Agent 还需要工具、流程、记忆、审核

> 绘本案例：DeepSeek V4 不会画画，但 Agent 通过调用图片 API 来画。模型是大脑，工具是手——大脑指挥手干活。

---

### 第 10 页：要素 ③ 流程（步骤）+ ④ 记忆（笔记本）

- **流程**：先做什么、后做什么、什么时候检查。绘本流程有严格的 6 步顺序
- **记忆**：记住前面做了什么，出第 6 页文案时知道第 1-5 页讲了什么
- 流程错了 = 翻车。比如"先生成插图再写文案"就反了

> 绘本案例：Agent 不会先画图再写文案——因为流程规定了顺序。也不会每页生成重复的内容——因为记忆让 Agent 知道前面写了什么。

---

### 第 11 页：要素 ⑤ 人工审核（把关）

- 关键决策不能交给 AI 自动完成
- 需要人工审核的场景：打分、发布给学生、学术内容、隐私和伦理
- Agent 的底线：**AI 干活，人把关**

> 绘本案例：大纲规划完→停，等你确认。网页生成完→停，等你预览。这两个点如果 AI 自己过了，可能整本绘本方向都偏了。

---

### 第 12 页：多智能体协作（拓展视野）

- 当一个 Agent 不够时，多个 Agent 可以协作
- 三种模式：
  - **老板-员工模式**：一个领导 Agent 分配任务，多个执行 Agent 干活
  - **流水线模式**：需求 → 架构 → 编码 → 测试 → 部署，每个环节一个 Agent
  - **互审模式**：多个 Agent 互相检查工作

> "今天下午只学单 Agent——一个 Agent 已经能做很多事了。多 Agent 是明天下午的内容。"
>
> ⚠️ **此页可压缩或跳过**（控制 Agent 理论在 30min 内）。如果超时，一句话带过："多个 Agent 可以协作，这是明天下午的内容。"

## 关键教学提示

1. **🔥 演示先行的关键**：开场 Demo 只展示效果、不讲概念。用"它是怎么做到的？"制造悬念，驱动 PPT 阶段的注意力。
2. **每个 PPT 页都要指回 Demo**。不要抽象讲概念。"刚才 Demo 里第 30 秒那个停顿，就是 Hook"。
3. **第 4 页的对比表**是整节课最重要的页。让教师直观感受"50 次操作 vs 2 次确认"的差距。可以在此处停顿 30 秒让教师消化。
4. **逐点对应改为问答互动**。不要独白——教师已经看了 Demo 又听了 PPT，让他们自己说出来，检验理解。
5. **第 12 页（多智能体）可压缩**。如果前面超时，一句话带过即可。
