# 教学机部署运行手册 — FLUX.2-klein-9B 推理服务

> **这份文档的用途**：在教学机上部署并启动 FLUX 文生图服务，供老师们在自己电脑上远程调用。
> 文档同时写给两类读者——① 帮忙操作教学机的 AI 助手（Claude），② 现场的老师。
> 每一步都给了「确切命令 + 成功长什么样 + 失败怎么办」，照着顺序走即可。

---

## 0. 场景与前提

### 机器角色

| 机器 | 角色 | 干什么 |
|------|------|--------|
| **教学机**（有 GPU） | 服务端 | 跑 `server.py`，加载模型，对外提供 API |
| **老师电脑**（局域网内） | 客户端 | 跑 `lan_test.py` 或用浏览器调用，出图 |

两台机器在**同一局域网**，通过教学机的内网 IP + 端口 `5500` 通信。

### 教学机环境（已确认）

- 操作系统：Windows 或 Linux 均可
- **CUDA 12.6**
- **已装 conda**（Anaconda / Miniconda）
- 有 NVIDIA GPU，显存建议 ≥ 24GB（9B 模型 BF16 约 18GB）
- 模型文件 **已下载好**（若没下，见 [第 3 步](#3-下载模型仅当模型不存在)）

### 镜像源策略（重要）

- **torch** → 南京大学 cu126 源（清华源没有 CUDA 版 torch）
- **其余所有包** → 清华源
- 这两个源已经写死在 [requirements.txt](requirements.txt) 顶部，`pip install -r requirements.txt` 会自动分流，**不用手动加 `-i`**。

## 1. 环境自检（先确认，再动手）

按顺序执行，四条都通过再进下一节。

```bash
# ① 确认 GPU 和驱动可见（会显示 CUDA 版本、显存）
nvidia-smi

# ② 确认 conda 可用
conda --version

# ③ 确认在服务代码目录里（能看到 server.py、requirements.txt）
#    Linux:  ls
#    Windows: dir
ls        # 应能看到 server.py  requirements.txt  lan_test.py

# ④ 确认模型已下载（能看到一堆 .safetensors 和 config）
#    路径以实际为准，默认是 ./models/FLUX.2-klein-9B
ls models/FLUX.2-klein-9B
```

**成功判据**：`nvidia-smi` 右上角显示 `CUDA Version: 12.6`（或更高），`conda --version` 有版本号，`ls` 能看到 `server.py`。

**失败分支**：
- `nvidia-smi` 找不到命令 / 无输出 → GPU 驱动没装好，**停下来**，本手册后续步骤没有意义。
- 第 ④ 步目录为空 → 模型没下，跳到 [第 3 步](#3-下载模型仅当模型不存在)。

---

## 2. 建环境 + 装依赖

### 2.1 建一个全新的 conda 环境

```bash
conda create -n flux python=3.11 -y
conda activate flux
```

> 固定用 **Python 3.11**：torch 2.8 / diffusers / transformers 对它支持最成熟，比 3.12/3.13 稳。
> 之后每次开新终端操作前，都要先 `conda activate flux`。

**成功判据**：命令行提示符前面出现 `(flux)`。

### 2.2 一键装依赖

```bash
pip install -r requirements.txt
```

这一步会下载 torch（约 2.5GB，走南大源）+ 其余包（走清华源），耗时几分钟到十几分钟，取决于网速。

**成功判据**：最后没有红色 `ERROR`，出现 `Successfully installed torch-2.8.0+cu126 ...` 等。

### 2.3 验证 torch 能用 GPU（关键一步，别跳过）

```bash
python -c "import torch; print(torch.__version__); print('CUDA 可用:', torch.cuda.is_available())"
```

**成功判据**：输出类似
```
2.8.0+cu126
CUDA 可用: True
```

**失败分支**：
- 打印 `CUDA 可用: False` → torch 没识别到 GPU。多半是装成了 CPU 版。重装：
  ```bash
  pip uninstall -y torch
  pip install torch==2.8.0+cu126 --index-url https://mirrors.nju.edu.cn/pytorch/whl/cu126
  ```
- 南大源下载慢/失败 → 换阿里云源：
  ```bash
  pip install torch==2.8.0+cu126 --index-url https://mirrors.aliyun.com/pytorch-wheels/cu126
  ```

### 2.4 确认 diffusers 支持 FLUX.2（可能需要补装 git 版）

```bash
python -c "from diffusers import Flux2KleinPipeline; print('OK: Flux2KleinPipeline 可用')"
```

**成功判据**：打印 `OK: Flux2KleinPipeline 可用` → 直接进第 4 步。

**失败分支**：报 `ImportError: cannot import name 'Flux2KleinPipeline'` → 稳定版还没有这个类，补装 GitHub 主分支版：
```bash
pip install "git+https://github.com/huggingface/diffusers.git" -i https://pypi.tuna.tsinghua.edu.cn/simple
```
装完再跑一次上面的验证命令确认。
> 若 GitHub 连不上，可加代理前缀，例如把 URL 换成 `git+https://ghfast.top/https://github.com/huggingface/diffusers.git`（代理站点会变，连不上就换一个）。

## 3. 下载模型（仅当模型不存在）

第 1 步第 ④ 项目录为空时才做。国内用 ModelScope 最快（约 18GB，10-30 分钟）：

```bash
pip install modelscope    # requirements 里已含，通常已装
modelscope download --model black-forest-labs/FLUX.2-klein-9B \
  --local_dir ./models/FLUX.2-klein-9B
```

**成功判据**：`./models/FLUX.2-klein-9B` 下出现多个 `.safetensors` 文件和 `model_index.json`。

---

## 4. 启动服务

### 4.1 启动命令（两种系统通用）

```bash
conda activate flux    # 确保在 flux 环境里
python server.py --model-path ./models/FLUX.2-klein-9B --host 0.0.0.0 --port 5500
```

- `--host 0.0.0.0` **必须有**，否则老师电脑连不进来。
- 模型加载要几分钟（9B + Qwen3 编码器）。

**成功判据**：日志出现
```
✅ 模型加载完成
INFO:     Uvicorn running on http://0.0.0.0:5500
```

**失败分支**：
- `CUDA out of memory` → 显存不够，改用 CPU offload 重启：
  ```bash
  python server.py --model-path ./models/FLUX.2-klein-9B --host 0.0.0.0 --port 5500 --cpu-offload
  ```

### 4.2 让服务在后台常驻（关掉终端也不停）

- **Linux**：用 tmux
  ```bash
  tmux new -s flux        # 进会话后再执行 4.1 的启动命令
  # 启动成功后按 Ctrl+B 松开再按 D → 退出但服务继续跑
  # 想回来看日志：tmux attach -t flux
  ```
- **Windows**：直接**保持那个终端窗口开着**别关，或用「开始菜单 → Anaconda Prompt」单独开一个窗口跑，最小化即可。

### 4.3 记下教学机 IP（老师电脑要用）

```bash
# Linux
hostname -I | awk '{print $1}'
# Windows
ipconfig        # 看 IPv4 地址，形如 192.168.x.x
```

把这个 IP 记下来，第 6 步和发给老师都用它。

---

## 5. 放行端口 5500

- **Linux（ufw）**：
  ```bash
  sudo ufw allow 5500/tcp
  ```
- **Windows 防火墙**：控制面板 → Windows Defender 防火墙 → 高级设置 → 入站规则 → 新建规则 → 端口 → TCP → 5500 → 允许连接。
- 教学机自测（在教学机本机上）：
  ```bash
  curl http://localhost:5500/health
  ```
  返回一段 JSON（含 `"status":"healthy"`）就说明服务本身没问题。

## 6. 老师电脑测试

在**老师电脑**上（不是教学机），进到同一份代码的 `flux-inference-service` 目录：

```bash
pip install requests     # 老师电脑只需要这一个包

# IP 换成第 4.3 步记下的教学机 IP
python lan_test.py --url http://192.168.x.x:5500
```

脚本会依次跑：健康检查 → 提交作业 → 等待生成 → 保存图片 `lan_test_output.png`。

也可以直接用浏览器打开队列看板：`http://192.168.x.x:5500/v1/queue`

**全流程成功判据**：老师电脑目录下出现 `lan_test_output.png`，能打开看到图。

## 7. 常见问题速查

| 现象 | 原因 | 处理 |
|------|------|------|
| 老师电脑 `无法连接` | 防火墙没放行 / 漏了 `--host 0.0.0.0` | 做第 5 步；确认启动命令带 `0.0.0.0` |
| `ping 教学机IP` 不通 | 不在同一局域网 | 检查网线/WiFi 是否同一网络 |
| `/health` 返回 `degraded` | 服务起了但模型没加载成功 | 看启动日志报错，多半显存不足，加 `--cpu-offload` |
| `CUDA 可用: False` | 装成了 CPU 版 torch | 按第 2.3 步失败分支重装 |
| `Flux2KleinPipeline` 导不进 | diffusers 版本旧 | 按第 2.4 步装 git 版 |
| torch 下载极慢 | 源不通 | 南大 ↔ 阿里云源互换（第 2.3 步） |
| SSH 断开后服务停了 | 没用 tmux | 按第 4.2 步用 tmux/独立窗口 |

## 8. 给操作 AI 助手的说明（务必遵守）

- **只在明确的一台机器上执行命令**：服务端命令跑在教学机，测试命令跑在老师电脑，别搞混。执行前先 `hostname` 确认当前在哪台机。
- **按本手册顺序走，每步先看「成功判据」再进下一步**；某步失败先走对应「失败分支」，不要跳步。
- **不做破坏性操作**：不要 `rm -rf`、不要删除 `models/` 下的模型文件、不要覆盖老师已有的 conda 环境。涉及删除/覆盖时，先向老师说明再执行。
- **模型文件很大（18GB）**：确认存在就直接用，**不要重复下载**。
- **需要 sudo / 管理员权限的操作**（放行防火墙等）先告知老师，由老师确认。
- 遇到本手册没覆盖的报错，**把完整报错贴出来给老师看**，不要自行猜测执行高风险命令。
